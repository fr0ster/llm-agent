# Extract OpenAI LLM and Embedder — Design Spec

**Date:** 2026-04-22
**Target release:** v10.1.0 (minor, additive)
**Status:** Draft → In Review
**Builds on:** v10.0.0 (monorepo split)

## Motivation

v10.0.0 split the repository into `@mcp-abap-adt/llm-agent` (core) and `@mcp-abap-adt/llm-agent-server` (runtime), but left LLM providers and embedders inside those two packages. A consumer who only needs one provider (e.g. DeepSeek LLM + Ollama embeddings, a pattern actually used by `examples/docker-deepseek/`) still pulls OpenAI-specific code.

v10.1.0 extracts the first provider family — OpenAI — into dedicated packages, proving the pattern that subsequent minor releases (10.2.0, 10.3.0, …) will repeat for every other provider. The goal is a strictly smaller runtime surface for consumers who don't use OpenAI.

Two separate packages are created (`openai-llm` + `openai-embedder`) rather than one, because the two artifacts are independently useful: a consumer can pair `openai-llm` with any embedder (Ollama, SAP AI Core, local) or pair `openai-embedder` with any LLM. Lump-ing them into one package forces dead weight on mixed-provider deployments.

## Scope

### In scope

- Create `packages/openai-llm/` with `OpenAIProvider` (implements `ILlm`).
- Create `packages/openai-embedder/` with `OpenAiEmbedder` (implements `IEmbedder`).
- Move the two source files into the new packages via `git mv`.
- Add the new packages as dependencies to `@mcp-abap-adt/llm-agent-server` (openai-llm) and `@mcp-abap-adt/llm-agent` (openai-embedder).
- Re-export `OpenAIProvider` from `@mcp-abap-adt/llm-agent-server`'s public surface for back-compat.
- Re-export `OpenAiEmbedder` from `@mcp-abap-adt/llm-agent`'s public surface for back-compat.
- Rewire `DeepSeekProvider` (stays in server) to import `OpenAIProvider` from `@mcp-abap-adt/openai-llm`.
- Update `embedder-factories.ts` (in core) to import `OpenAiEmbedder` from `@mcp-abap-adt/openai-embedder`.
- Update the `fixed` changesets group to include the four packages so all four stay in lock-step for the 10.x lifecycle.
- Bump `llm-agent`, `llm-agent-server` to 10.1.0. Initial `openai-llm`, `openai-embedder` release at 10.1.0.
- Docs: add `MIGRATION-v10.1.md` noting the new canonical import paths and explaining that old paths still work.

### Out of scope

- Other LLM providers (Anthropic, DeepSeek, SAP AI Core) — planned for 10.2.0+.
- Other embedders (Ollama, SAP AI Core) — planned for 10.3.0+.
- QdrantRag extraction — planned for 10.6.0 (closes the last heavy runtime dep in core).
- Any behavioral change to `OpenAIProvider` or `OpenAiEmbedder` — no logic edits.
- Removing deprecated re-exports — future major bump (v11).

## Changesets configuration

### Update `.changeset/config.json`

```json
{
  "fixed": [
    [
      "@mcp-abap-adt/llm-agent",
      "@mcp-abap-adt/llm-agent-server",
      "@mcp-abap-adt/openai-llm",
      "@mcp-abap-adt/openai-embedder"
    ]
  ]
}
```

All four packages stay in lock-step through 10.x. Any non-trivial release bumps all four, even if the change touches only one. Rationale: the four packages form a tight cluster where internal deps are pinned to `*`. Version drift risks silent breakage; lock-step is safe.

When future providers are extracted (10.2.0+), they join the fixed group as well.

### Changeset file for this release

`.changeset/10.1-extract-openai.md`:

```markdown
---
"@mcp-abap-adt/llm-agent": minor
"@mcp-abap-adt/llm-agent-server": minor
---

Extract OpenAI LLM and embedder into dedicated packages.

New packages (shipped at 10.1.0):
- `@mcp-abap-adt/openai-llm` — OpenAIProvider (LLM).
- `@mcp-abap-adt/openai-embedder` — OpenAiEmbedder.

Existing import paths keep working via re-exports. Consumers who want a lighter dependency footprint can import directly from the new packages.

DeepSeekProvider stays in `@mcp-abap-adt/llm-agent-server` and now imports OpenAIProvider from the new package transparently.
```

Both new packages don't need their own entries because changesets auto-adds them to the bump list via the `fixed` group once they appear in workspaces with a populated `package.json`. First publish at 10.1.0.

## Release flow

1. Implement changes (Tasks 1-6 in the plan).
2. `npx changeset` — add the changeset file documented above.
3. `npx changeset version` — bumps all four packages to 10.1.0, writes per-package CHANGELOG.md, removes the changeset file.
4. `npm run build && npm test` — verify.
5. Commit bump, merge PR.
6. Post-merge: `npx changeset publish` publishes all four packages in dependency order in one command.
7. Tag `v10.1.0`, push — GitHub release workflow creates the release page.

## Migration for consumers — `docs/MIGRATION-v10.1.md`

Short doc (≤ 40 lines). Contents:

- **What's new:** two dedicated packages for OpenAI LLM and embedder.
- **What stays the same:** all existing import paths still work — re-exports preserve v10.0 surface.
- **Lighter deps example:** "If you use DeepSeek LLM and Ollama embeddings, you can now write `import { OllamaEmbedder } from '@mcp-abap-adt/llm-agent'` without pulling OpenAI code — install only `@mcp-abap-adt/llm-agent-server @mcp-abap-adt/ollama-embedder` (when available)."
- **New canonical paths:**
  - `import { OpenAIProvider } from '@mcp-abap-adt/openai-llm'`
  - `import { OpenAiEmbedder } from '@mcp-abap-adt/openai-embedder'`
- **Deprecation note:** old paths not deprecated in 10.1. Removed in a future major.
- **Roadmap reminder:** other providers follow in 10.2, 10.3, 10.4.

## Known debt

- `@mcp-abap-adt/llm-agent` still carries `axios` transitively through `@mcp-abap-adt/openai-embedder` (because core re-exports OpenAiEmbedder). That's not meaningful progress on "lightweight core" for consumers who install core alone. Real dep shrinkage happens when `openai-embedder` becomes optional (consumers pick only providers they need) — which is a v11 breaking change to remove the re-exports.
- Workspace publish order depends on `changeset publish` reading the `dependencies` graph correctly. If it doesn't, manual ordering: `openai-llm` and `openai-embedder` first, then core, then server.
