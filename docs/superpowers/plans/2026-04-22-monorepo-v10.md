# Monorepo Restructure v10.0.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert the single-package repo into an npm-workspaces monorepo; split code into `@mcp-abap-adt/llm-agent` (core: interfaces + lightweight RAG impls) and `@mcp-abap-adt/llm-agent-server` (default SmartAgent, pipeline, LLM providers, MCP client, HTTP server, CLIs). Ship v10.0.0 with lock-step versioning via Changesets.

**Architecture:** In-place `git mv` migration. Root `package.json` becomes workspace root with `"private": true`. Core and server each build via `tsc` with TypeScript project references. Biome config stays at root. Three bins (`llm-agent`, `llm-agent-check`, `claude-via-agent`) all live in server package.

**Tech Stack:** TypeScript strict, ESM, npm workspaces, `@changesets/cli`, Biome, `node:test` via `tsx`.

**Spec:** `docs/superpowers/specs/2026-04-22-monorepo-v10-design.md`

**Branch:** `feat/monorepo-v10`

**Nature of this plan:** migration, not TDD. "Test" for each task = existing test suite still passes + build is green. No new behavior is added.

## File map overview

### What moves to `packages/llm-agent/` (core)
- `src/smart-agent/interfaces/` → `packages/llm-agent/src/interfaces/`
- `src/smart-agent/rag/` → `packages/llm-agent/src/rag/`
- Subset of `src/smart-agent/interfaces/*.ts` that references only core-local types

### What moves to `packages/llm-agent-server/` (server)

- `src/smart-agent/{agent,builder,cli,check-models-cli,smart-server,server,config,providers,pipeline,mcp-client-factory}.ts` → `packages/llm-agent-server/src/smart-agent/`
- `src/smart-agent/{adapters,api-adapters,cache,classifier,config,context,health,history,logger,metrics,otel,pipeline,plugins,policy,reranker,resilience,session,skills,testing,tracer}/` → `packages/llm-agent-server/src/smart-agent/`
- `src/llm-providers/` → `packages/llm-agent-server/src/llm-providers/`
- `src/mcp/` → `packages/llm-agent-server/src/mcp/`
- `src/agents/` → `packages/llm-agent-server/src/agents/` (if present)
- `src/types.ts` → decided case-by-case: generic types (`Message`, `ToolCall`, `LLMResponse`) go to core; server-only types stay with server. Plan Task 5 makes the final call.
- `src/index.ts` → split into `packages/llm-agent/src/index.ts` and `packages/llm-agent-server/src/index.ts`
- `src/smoke-adapters.ts`, `src/generate-schemas.ts` → `packages/llm-agent-server/src/` (tooling, not public API)
- `bin/` (if present) → `packages/llm-agent-server/bin/`
- `tools/claude-via-agent.{sh,ps1}` → `packages/llm-agent-server/tools/`
- `scripts/generate-version.js` → `packages/llm-agent-server/scripts/`

### What stays at repo root
- `docs/` (all user-facing docs are repo-level)
- `examples/` (not published)
- `biome.json` (shared config)
- `README.md` (repurposed as monorepo index)
- `CHANGELOG.md` (monorepo-level aggregate; per-package changelogs auto-generated by Changesets)
- `LICENSE`
- `.github/workflows/`

- [ ] **Step 5: Convert root `package.json` to workspace root**

Current root `package.json` publishes as `@mcp-abap-adt/llm-agent`. We need to rename it to a private monorepo root (we'll create the real package under `packages/llm-agent/` in Task 2). Edit:

```json
{
  "name": "llm-agent-monorepo",
  "private": true,
  "version": "0.0.0",
  "workspaces": ["packages/*"],
  "scripts": {
    "build": "tsc -b packages/llm-agent packages/llm-agent-server",
    "clean": "tsc -b --clean packages/llm-agent packages/llm-agent-server",
    "lint": "biome check --write packages scripts docs examples",
    "lint:check": "biome check packages scripts docs examples",
    "format": "biome format --write packages scripts docs examples",
    "test": "npm run test --workspaces --if-present",
    "changeset": "changeset",
    "version": "changeset version",
    "release": "npm run build && changeset publish"
  },
  "devDependencies": {
    "@biomejs/biome": "^2.4.9",
    "@changesets/cli": "^2.x",
    "typescript": "^5.x",
    "tsx": "^4.x",
    "rimraf": "^5.x"
  }
}
```

**Important:** remove `main`, `types`, `exports`, `bin`, `files`, `dependencies`, `peerDependencies`, `peerDependenciesMeta`, and the publish-time `publishConfig` — those belong to individual packages now, not to the root. Preserve other fields (description, author, license, repository, etc.) or move them to per-package `package.json`s in Tasks 2/3.

Move `dependencies` and `peerDependencies` content aside — you'll put them into the right per-package `package.json`s in Tasks 2/3. Also move `prebuild` / `smoke:adapters` / `test:adapters` / `test:server` scripts aside — those reference `src/` paths that won't exist; reintroduce them inside the server package in Task 3.
